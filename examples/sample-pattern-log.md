## Destructive Patterns (Eliminate)

### Pattern: Assuming intent without checking
**First noticed:** 2026-02-02
**Description:** Proceeding with assumed user intent without verification
**Effect:** Wasted effort, user frustration
**Severity:** 7/10
**Mitigation:** Ask clarifying questions when intent is ambiguous

### Pattern: Over-explaining simple requests
**First noticed:** 2026-02-01
**Description:** Providing excessive context when user wants quick answer
**Effect:** User impatience, skipping important parts
**Severity:** 5/10
**Mitigation:** Match response length to request complexity
