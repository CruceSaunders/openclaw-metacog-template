# File Audit Checklist

_Use this to review workspace files for coherence, accuracy, and completeness._

---

## Quick Coherence Check (5 minutes)

Run this weekly or when something feels "off":

### SOUL.md
- [ ] Does it still describe who I am accurately?
- [ ] Are the core truths still true based on experience?
- [ ] Are there new principles that should be added?
- [ ] Is anything outdated or no longer relevant?

### AGENTS.md  
- [ ] Do the operational instructions match my actual practice?
- [ ] Are there processes I follow that aren't documented?
- [ ] Are there documented processes I've stopped following?
- [ ] Do the rules make sense given recent learnings?

### MEMORY.md
- [ ] Is the information current?
- [ ] Are there old beliefs that should be updated?
- [ ] Are recent lessons properly logged?
- [ ] Is there stale information that should be removed?

### Skills
- [ ] Are all skills I use documented?
- [ ] Are skill instructions accurate and complete?
- [ ] Are there skills that need updates based on experience?

## Red Flags to Watch For

### üö© Contradictory Instructions
**Example:** SOUL.md says "be bold" but AGENTS.md says "always ask first"
**Fix:** Resolve the contradiction with clear rules about when each applies

### üö© Outdated Information
**Example:** MEMORY.md lists a belief you've already disproven
**Fix:** Update with current understanding, mark old belief as superseded

### üö© Missing Documentation
**Example:** You follow a process that isn't written anywhere
**Fix:** Document it so future sessions have access

### üö© Dead Rules
**Example:** A rule in AGENTS.md that you never actually follow
**Fix:** Either start following it or remove/update it

### üö© Conflicting Learnings
**Example:** Two lessons in MEMORY.md that suggest opposite approaches
**Fix:** Reconcile with more specific guidance about when each applies
